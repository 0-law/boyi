# 公告处理规则

## 适用范围
适用于上市公司公告、交易所文件、监管问询回复、定期报告、业绩预告、中标订单公告、澄清公告、减持公告、停复牌公告等。

## 基本原则
1. 公告属于高优先级信息源，是公司事实判断的核心依据。
2. 不能只看标题，必须看正文、金额、对象、条件、风险提示。
3. 公告中的事实、前提条件、市场解读要分开。

## 处理步骤

### 第一步：识别公告类型
- 定期报告
- 业绩预告 / 快报
- 重大合同 / 中标
- 投资扩产
- 并购重组
- 澄清公告
- 减持 / 质押 / 回购
- 监管关注 / 问询回复

### 第二步：抽取关键事实
至少提取以下字段：
- 公告时间
- 发布主体
- 事项名称
- 涉及金额或规模
- 对手方或项目方
- 生效条件
- 是否存在重大不确定性

### 第三步：区分公告中的三层内容
- 事实：公告明确写明的内容
- 推演：市场据此联想到的弹性空间
- 风险：公告中提示的落地条件、履约风险、审批风险

## 重点公告识别

### 强事实公告
- 业绩预告
- 中标 / 重大合同公告
- 澄清公告
- 定期报告
- 回购公告
- 监管正式文件

处理要求：默认 H5，可直接作为事实依据；但仍需单独评估 L / C / I，不能自动等于 SS 或 SSS。

### 需谨慎解读公告
- 框架协议
- 战略合作
- 意向书
- 项目中标候选人
- “拟”投资、“拟”收购

处理要求：事实层面仍可记录，但交易结论不能等同于已经落地。

## 高风险误判点
- 将“中标候选人”误判成正式中标
- 将“签署框架协议”误判成确认收入
- 将“预计”误判成“已经实现”
- 忽略订单金额占营收比例
- 忽略项目执行周期与兑现节奏

## 输出建议
- 确认落地、金额明确、条款清晰的公告，可作为“主攻支撑”或“弹性支撑”。
- 不确定性高、条款模糊的公告，应标记验证点并列入“观察”。
- 针对传闻的澄清公告，优先用于“降级”或“排雷”。
