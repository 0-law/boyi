# 真实资讯测试规则

## 目标

本文件用于真实文章、复盘文章、盘中热点汇总、短评、研报摘录、小作文合集的测试分析。

目标不是给交易建议，而是检验“资讯裁判模块”是否能稳定完成：

1. 逐条资讯拆分
2. 来源分类
3. 主体类型识别
4. A股映射链识别
5. 事实 / 观点 / 推演 / 传闻 / 情绪 / 风险拆分
6. 时间字段读取与继承
7. `source_session / ingest_session / timeliness_status` 判断
8. H / L / C / W / I 评级
9. 验证点与证伪触发点提取
10. 逻辑疑点识别
11. 降级与排雷
12. 输出统一表格
13. 输出可回放的自检结果

采集输入格式详见：[14_采集输入格式规范.md](./14_采集输入格式规范.md)

---

## 时间处理边界

1. 统一使用 `ISO 8601 + UTC+08`，默认时区 `Asia/Shanghai`。
2. 原文只有日期时，只保留日期，不得补具体时分秒。
3. 原文没有发布时间时，`source_time` 写 `null`，`source_time_precision` 写 `unknown`。
4. `source_session` 反映资讯原始发生时段，不得被 `ingest_time` 覆盖。
5. `ingest_session` 只反映系统何时拿到资讯。
6. `timeliness_status` 只使用 `实时 / 延迟获取 / 旧闻重抓 / 已过催化窗口 / 待盘面验证 / unknown`。
7. 没有盘面数据时，不得写 `已兑现 / 已部分兑现 / 高开低走 / 资金承接强 / 盘面证伪`。
8. 时间不会改变 H 证据硬度，只影响 C / W / I、时效状态和处理结果。

## 输入要求

输入可以是完整文章，也可以是多条混合资讯。

输入中可能包含：

- 公告
- 新闻
- 政策
- 研报观点
- 纪要
- 复盘信息
- 小作文
- 传闻
- 自媒体观点
- 涨停原因
- 异动原因
- OCR 文本
- AI 摘要
- 个股逻辑
- 板块逻辑
- 海外公司新闻
- 未上市公司新闻
- 产品模型发布
- 时间节点 / 事件日历
- 市场情绪描述

## 逐条资讯拆分规则

一条独立催化必须单独成行。

不得为了压缩输出，把不同来源、不同主体、不同证据硬度、不同证伪窗口、不同风险结构的资讯合并成一行。

以下情况必须拆分：

1. 同一板块下包含多个不同事件。
2. 同一方向下同时出现公告、新闻、小作文、复盘观点。
3. 同一段文字里同时包含事实、观点、推演、传闻。
4. 同一链条里同时包含海外公司、A 股公司、未上市公司、政策机构、产品模型等不同主体。
5. 同一主题下某条为 H5，另一条为 H1 / H2。
6. 同一主题下某条已被证伪，另一条仍处于观察状态。

---

## 主体类型规则

资讯裁判必须区分：

- A股上市公司
- 港股上市公司
- 美股上市公司
- 海外公司
- 未上市公司
- 政策/监管机构
- 行业协会/交易所
- 产品/模型/项目
- 行业方向
- 宏观数据
- 未知主体

非 A 股主体不得直接写成 A 股股票。

如需映射 A 股，必须标注“映射链待验证”。

---

## 信息性质约束

以下措辞不得直接判定为事实：

- 预计
- 有望
- 或将
- 拟
- 计划
- 据悉
- 消息称
- 知情人士
- 供应商
- 网传
- 爆料
- 小作文
- 大V

约束规则：

1. `预计 / 有望 / 或将` 优先标记为 `预期` 或 `待验证事实`。
2. `拟 / 计划` 优先标记为 `计划`。
3. `据悉 / 消息称 / 知情人士 / 供应商` 优先标记为 `媒体消息 / 行业消息 / 待验证事实`。
4. `网传 / 爆料 / 小作文 / 大V` 优先标记为 `传闻 / 观点 / 自媒体消息`。
5. 只有正式公告、监管文件、定期报告、业绩预告、正式政策文件等，才能稳态归为事实。

---

## 资讯侧支持边界

“主攻支撑”和“弹性支撑”只表示资讯侧支持，不代表最终交易结论。

在没有接入以下信息前：

- 板块资金流入
- 个股资金流入
- 成交额
- 换手率
- 分时承接
- 涨停数量
- 连板高度
- 龙头地位
- 板块扩散
- 高位股风险

不得把“主攻支撑 / 弹性支撑”解释为交易主攻、买入、加仓、目标价或持仓建议。

## 回归测试清单

使用当前样例与后续真实文章测试时，至少逐项检查以下问题：

1. `对应个股` 中不能出现 `[锐捷网络]()` 这类空链接。
2. 全文不能出现 `C:\Users` 这类本地绝对路径。
3. 样例中 `source_session` 不能因为 `ingest_time` 在周末就被一律写成 `非交易日`。
4. “预计 7 月 6 日上线”不能被写成纯事实。
5. “拟于 7 月推出 / 计划部署”不能被写成纯事实。
6. 没有盘面数据时，不能输出 `已部分兑现`。
7. 海外主体在没有 A 股验证前不能直接给 `SS`。
8. 自检中不能写“所有主体已覆盖”这种无依据结论，必须改为 `是 / 否 + 说明`。
9. 宏观、政策或行业文章如果存在逻辑跳跃，必须写入 `原文逻辑疑点 / 疑点说明`。
10. 小作文高 W 但低 L 的场景，不能因为“不可证伪”而升级。
11. 交易主体与背景主体必须拆表。
12. 对应个股展示必须统一成“名称（代码/状态）”。

## 测试合格标准

1. 所有独立催化都单独成行。
2. 非 A 股主体不得混入 A 股股票。
3. 未能唯一确认代码时不得强补代码。
4. 原文未明确公告时，不得直接标为公告 / H5。
5. 小作文不得当事实。
6. 复盘观点不得当公司事实。
7. 每条高风险资讯都必须给出证伪触发点。
8. 每个非 A 股主体都必须标注主体类型。
9. 每个非 A 股主体若涉及 A 股映射，必须标注“映射链待验证”。
10. 不得输出买卖建议、目标价、仓位建议。
11. 不得缺失统一时间字段组。
12. 不得残留 Markdown 链接污染、本地路径污染、空链接污染。
