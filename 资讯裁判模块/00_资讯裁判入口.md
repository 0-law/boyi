# 资讯裁判模块入口

## 模块目标
本模块用于处理 A 股短线投研中的资讯类内容，统一对公告、新闻、政策、研报、纪要、复盘、小作文、传闻、自媒体、图片识别文字、AI 摘要等信息进行裁判、分级、降级与输出。

## 核心原则
1. 资讯不能直接等于事实，必须先裁判再使用。
2. 必须区分事实、观点、推演、传闻、情绪、风险。
3. 必须区分证据硬度、逻辑合理性、环境适配度、证伪窗口、影响力预期与最终短线评级。
4. 小作文不能当事实，但可作为资金攻击线索观察。
5. 公告、监管文件、业绩预告、中标订单、澄清公告属于强事实。
6. 复盘、自媒体、盘前纪要主要用于识别市场共识，不得直接当作公司事实。
7. AI 总结不是证据，只能作为辅助整理。
8. 图片识别文字默认低置信，除非有明确来源。
9. 与公司公告或监管信息冲突的传闻，必须降级处理。
10. 所有资讯最终统一输出为“资讯裁判表”。
11. 真实性低不等于影响力低。
12. 真实性高不等于短线影响力高。
13. 公告可以提高证据硬度，但不必然提高短线影响力。
14. 最终评级 SSS/SS/S/A/B/C/R 表示短线博弈影响力，不表示事实真实性。
15. 信息的短线影响力，不只取决于真假，也取决于它在多长时间内不会被证伪。
16. 小作文默认证据硬度低，但如果具备较长证伪窗口，短线影响力可能显著放大。
17. 证伪窗口越长，资金越容易围绕该信息进行叙事扩散。
18. 证伪窗口只提高博弈空间，不提高真实性。
19. 不可证伪不等于有价值，必须同时评估 L 逻辑合理性和 C 环境适配度。
20. 一旦出现澄清、监管、公告冲突、权威核查或盘面快速否定，证伪窗口关闭，最终评级必须快速降级。

## 使用顺序
1. 原始资讯输入。
2. 资讯源分类。
3. 事实 / 观点 / 推演 / 传闻 / 情绪 / 风险拆分。
4. 板块与股票抽取。
5. H 证据硬度评级。
6. L 逻辑合理性评级。
7. C 环境适配度评级。
8. W 证伪窗口评级。
9. I 影响力预期评级。
10. 最终评级 SSS / SS / S / A / B / C / R。
11. 输出资讯裁判表。
12. 漏股、漏资讯、证伪风险自检。

## 裁判流程
### 第一步：识别来源
- 公告类
- 新闻类
- 政策类
- 研报纪要类
- 复盘类
- 小作文与传闻类
- 自媒体类
- 数据异动类
- OCR 与 AI 二次整理类

### 第二步：执行多维评级
按照 [02_证据等级规则.md](./02_证据等级规则.md) 执行，统一采用 H / L / C / W / I + 最终评级。

### 第三步：拆分内容结构
每条资讯至少拆成以下部分：
- 可验证事实
- 主体观点
- 市场推演
- 情绪表达
- 潜在风险

### 第四步：执行降级与排雷
遇到来源不明、图片截图、二手转述、与公告冲突、时间过期、标题党等情况，必须参考 [09_降级与排雷规则.md](./09_降级与排雷规则.md) 进行降级。

### 第五步：统一输出
所有内容最终落到 [08_资讯输出模板.md](./08_资讯输出模板.md) 的“资讯裁判表”中，不允许只保留原文而不裁判。

## 最低落地要求
- 每条资讯必须有来源类型。
- 每条资讯必须有 H / L / C / W / I 与最终评级。
- 每条资讯必须标记信息性质。
- 每条资讯必须写明验证点。
- 每条资讯必须给出处理结果。

## 模块定位
本基础版只提供规则文件和模板文件，不接数据库，不做自动打分程序，不替代研究员判断，只为后续标准化录入与人工裁判提供统一框架。
