# 资讯裁判模块入口

## 模块目标

本模块用于处理 A 股短线投研中的资讯类内容，统一对公告、新闻、政策、研报、纪要、复盘、小作文、传闻、自媒体、图片识别文字、AI 摘要等信息进行裁判、分级、降级与输出。

---

## 核心原则

1. 资讯不能直接等于事实，必须先裁判再使用。
2. 必须区分事实、观点、推演、传闻、情绪、风险。
3. 必须区分证据硬度、逻辑合理性、环境适配度、证伪窗口、影响力预期与最终短线评级。
4. 小作文不能当事实，但可作为资金攻击线索观察。
5. 公告、监管文件、业绩预告、中标订单、澄清公告属于强事实。
6. 复盘、自媒体、盘前纪要主要用于识别市场共识，不得直接当作公司事实。
7. AI 总结不是证据，只能作为辅助整理。
8. OCR 和图片识别文字默认低置信，除非有明确来源。
9. 与公司公告或监管信息冲突的传闻，必须降级处理。
10. 所有资讯最终统一输出到“资讯裁判结果”。
11. 时间不会改变 H 证据硬度，只影响 C、W、I、时效状态和处理结果。
12. 没有盘面数据时，不得输出已兑现、已部分兑现、高开低走、放量滞涨等盘面结论。

---

## 输入格式

本模块支持 Markdown 格式资讯输入。

采集端输入规范详见：

[14_采集输入格式规范.md](./14_采集输入格式规范.md)

资讯裁判模块只读取采集端提供的：

```text
source_time
source_time_precision
source_session
ingest_time
ingest_session
analysis_time
timezone
timeliness_status
```

不得编造时间。

---

## 使用顺序

1. 读取 Markdown 文档级元数据。
2. 读取每条资讯 YAML 元数据。
3. 判断 `source_time / source_time_precision / source_session / ingest_time / ingest_session / analysis_time / timezone / timeliness_status`。
4. 资讯源分类。
5. 独立催化拆分。
6. 事实 / 观点 / 推演 / 传闻 / 情绪 / 风险拆分。
7. 板块与题材抽取。
8. 主体全量抽取，区分交易主体与背景主体。
9. 主体类型识别。
10. A 股映射链判断。
11. H 证据硬度评级。
12. L 逻辑合理性评级。
13. C 环境适配度评级。
14. W 证伪窗口评级。
15. I 影响力预期评级。
16. 最终评级 `SSS / SS / S / A / B / C / R`。
17. 输出资讯裁判表。
18. 输出交易主体映射表。
19. 输出降级 / 排雷表。
20. 输出背景主体 / 证据主体表。
21. 执行漏股、漏资讯、时间字段、证伪风险自检。

## 裁判流程

### 第一步：识别来源

- 公告类
- 新闻类
- 政策类
- 研报纪要类
- 复盘类
- 小作文与传闻类
- 自媒体类
- 数据异动类
- OCR 或 AI 二次整理类

### 第二步：执行多维评级

按照 [02_证据等级规则.md](./02_证据等级规则.md) 执行，统一采用 `H / L / C / W / I + 最终评级`。

### 第三步：拆分内容结构

每条资讯至少拆成以下部分：

- 可验证事实
- 主体观点
- 市场推演
- 情绪表达
- 潜在风险

### 第四步：执行降级与排雷

遇到来源不明、图片截图、二手转述、与公告冲突、时间过期、标题党、空泛映射等情况，必须参考 [09_降级与排雷规则.md](./09_降级与排雷规则.md) 进行降级。

### 第五步：统一输出

所有内容最终落到 [08_资讯输出模板.md](./08_资讯输出模板.md) 的统一表格中，不允许只保留原文而不裁判。

---

## 最低落地要求

- 每条资讯必须有来源类型。
- 每条资讯必须有统一时间字段组。
- 每条资讯必须有 H / L / C / W / I 与最终评级。
- 每条资讯必须标记信息性质、验证点、证伪触发点、原文逻辑疑点和处理结果。
- 必须拆分交易主体映射表与背景主体 / 证据主体表。

---

## 模块定位

本基础版只提供规则文件、模板文件、样例与测试规则，不接数据库，不做自动打分程序，不替代研究员判断，只为后续标准化录入、可回放复核和人工裁判提供统一框架。
