# 资讯输出模板

## 统一输出原则
所有资讯最终输出为“资讯裁判表”，不允许只存原文，不允许跳过多维评级与处理结果。

## 资讯裁判表

## 字段说明

### 来源类型：
公告 / 新闻 / 政策 / 研报 / 复盘 / 小作文 / 自媒体 / 数据异动 / OCR低置信

### 信息性质：
事实 / 观点 / 推演 / 传闻 / 情绪 / 风险

### 催化类型：
政策 / 涨价 / 订单 / 业绩 / 供需缺口 / 技术突破 / 国产替代 / 并购 / 澄清 / 监管 / 减持 / 情绪扩散

### 影响级别：
板块级 / 个股级 / 情绪级 / 风险级

### 处理结果：
主攻支撑 / 弹性支撑 / 观察 / 高影响低可信观察 / 小作文池 / 需要公告验证 / 降级 / 排雷 / 仅记录

### 证伪触发点：
公司澄清 / 监管提示 / 交易所问询 / 董秘互动否认 / 权威媒体核查 / 客户供应商名单证伪 / 订单数据证伪 / 财报证伪 / 产品价格数据证伪 / 板块高开低走 / 核心股放量滞涨 / 资金撤退 / 市场风格切换

## 填写要求
1. 时间必须尽量精确到发布日期或传播时间。
2. 来源类型必须按规则归类，不得写模糊描述。
3. H / L / C / W / I 与最终评级必须引用 [02_证据等级规则.md](C:\Users\Administrator\Documents\金融投研博弈系统\资讯裁判模块\02_证据等级规则.md)。
4. 摘要只保留关键信息，不堆砌原文。
5. 对应方向可填写行业、题材、政策方向或市场主线。
6. 对应个股可填核心标的、跟风标的或留空。
7. 验证点必须可执行，不能写空泛表述。
8. 证伪触发点必须写明哪些事件会关闭叙事窗口。
9. 降级风险必须写明为何可能失真或失效。
10. 处理结果必须明确，不允许“待看”这类无裁判结论词。
11. 最终评级表示短线博弈影响力，不表示事实真实性。
