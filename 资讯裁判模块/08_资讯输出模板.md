# 资讯输出模板

采集输入格式详见：[14_采集输入格式规范.md](./14_采集输入格式规范.md)

## 统一输出原则

所有资讯最终输出为“资讯裁判结果”，至少包含：

1. 资讯裁判表
2. 交易主体映射表
3. 降级 / 排雷表
4. 背景主体 / 证据主体表
5. 自检结果

> 注意：本模板中的“主攻支撑 / 弹性支撑”仅表示资讯侧支持，不代表最终交易结论。最终交易判断需另行接入资金、盘面、技术结构和风险偏好。

## 字段说明

### 时间字段

- `source_time`：资讯原始发布时间，缺失时写 `null`。
- `source_time_precision`：只使用 `minute / hour / daypart / date / unknown`。
- `source_session`：只使用 `盘前 / 盘中 / 盘后 / 夜间 / 非交易日 / unknown`。
- `ingest_time`：系统获得 / 抓取 / 导入资讯的时间。
- `ingest_session`：只使用 `交易日盘前 / 交易日盘中 / 交易日盘后 / 非交易日 / unknown`。
- `analysis_time`：AI 实际裁判时间，缺失时写 `null`。
- `timezone`：统一写 `Asia/Shanghai`，除非原始输入另有明确时区。
- `timeliness_status`：只使用 `实时 / 延迟获取 / 旧闻重抓 / 已过催化窗口 / 待盘面验证 / unknown`。

时间规则：

1. `source_session` 反映资讯原始发生时段，不得被 `ingest_time` 覆盖。
2. `ingest_session` 只反映系统何时拿到该资讯。
3. `source_time` 缺失时，不得硬补具体时分秒。
4. `7月2日消息` 应写 `source_time_precision = date`。
5. `7月2日晚` 应写 `source_time_precision = daypart`，并将 `source_session` 识别为 `盘后` 或 `夜间`。
6. `7月2日7时46分` 应写 `source_time_precision = minute`。
7. 如果催化窗口已明显过去，可将 `timeliness_status` 标为 `已过催化窗口`。
8. 没有盘面数据时，不得将时效状态写成已兑现或已部分兑现。

### 来源类型

公告 / 新闻 / 政策 / 研报 / 复盘 / 小作文 / 自媒体 / 数据异动 / OCR低置信 / 时间节点 / 事件日历

### 对应个股

1. 统一写成“名称（代码/状态）”。
2. 不得出现 Markdown 链接、本地路径、空链接。
3. 示例：
   `锐捷网络（301165）`
   `昆仑万维（300418）`
   `宇树科技（未上市）`
   `苹果（美股，A股映射待验证）`
4. 明确识别为 A 股上市公司的，可补齐唯一 6 位代码；无法唯一确认时留空或写状态，不得臆造。

### 主体类型

A股上市公司 / 港股上市公司 / 美股上市公司 / 海外公司 / 未上市公司 / 政策/监管机构 / 行业协会/交易所 / 产品/模型/项目 / 行业方向 / 宏观数据 / 未知主体

### 信息性质

公告事实 / 政策事实 / 待验证事实 / 计划 / 预期 / 媒体消息 / 行业消息 / 观点 / 推演 / 传闻 / 情绪 / 风险

信息性质判定规则：

1. `预计 / 有望 / 或将` 优先归为 `预期` 或 `待验证事实`。
2. `拟 / 计划` 优先归为 `计划`。
3. `据悉 / 消息称 / 知情人士 / 供应商` 优先归为 `媒体消息`、`行业消息` 或 `待验证事实`。
4. `网传 / 爆料 / 小作文 / 大V` 不得直接标为事实，优先归为 `传闻`、`观点` 或 `自媒体消息`。
5. 只有正式公告、监管文件、定期报告、业绩预告、正式政策原文等，才能稳态归为 `公告事实` 或 `政策事实`。

### 催化类型

政策 / 涨价 / 订单 / 业绩 / 供需缺口 / 技术突破 / 国产替代 / 并购 / 澄清 / 监管 / 减持 / 情绪扩散

### 影响级别

板块级 / 个股级 / 情绪级 / 风险级

### 证伪触发点

公司澄清 / 监管提示 / 交易所问询 / 董秘互动否认 / 权威媒体核查 / 客户供应商名单证伪 / 订单数据证伪 / 财报证伪 / 产品价格数据证伪 / 板块高开低走 / 核心股放量滞涨 / 资金撤退 / 市场风格切换

### 原文逻辑疑点

填写 `有 / 无`。

### 疑点说明

用于记录原文中的逻辑跳跃、主体错配、因果倒置、时间矛盾、概念偷换、将预期写成事实等问题。

### 处理结果

主攻支撑 / 弹性支撑 / 观察 / 高影响低可信观察 / 小作文池 / 需要公告验证 / 降级 / 排雷 / 仅记录

边界规则：

1. `主攻支撑` 和 `弹性支撑` 只表示资讯侧支持。
2. 没有盘面数据时，不得输出“已兑现、已部分兑现、高开低走、资金承接强”等盘面判断。
3. 没有盘面数据时，处理结果应优先使用 `观察 / 高影响低可信观察 / 小作文池 / 需要公告验证 / 仅记录`。
4. 海外主体、未上市主体、海外政策或产业新闻，在没有明确 A 股订单、公告、业绩或盘面验证前，不得直接写成强交易结论。
