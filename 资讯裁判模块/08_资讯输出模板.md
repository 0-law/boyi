# 资讯输出模板

## 统一输出原则
所有资讯最终输出为“资讯裁判表”，不允许只存原文，不允许跳过多维评级与处理结果。

> 注意：本表中的“主攻支撑 / 弹性支撑”仅表示资讯侧支持，不代表最终交易结论。最终交易判断需要另行接入资金、盘面、技术结构和风险偏好。

## 字段说明

### 时间字段
- `source_time`：资讯原始发布时间，缺失时写 `null`。
- `source_time_precision`：原始时间精度，固定使用 `second / minute / hour / date / daypart_morning / daypart_noon / daypart_afternoon / daypart_night / range / unknown`。
- `ingest_time`：系统获得 / 抓取 / 导入资讯的时间。
- `analysis_time`：AI 实际裁判资讯的时间，缺失时写 `null`。
- `timezone`：统一写 `Asia/Shanghai`。

### 时段属性
盘前 / 盘中 / 盘后 / 非交易日 / 无法判断

判断优先级：

```text
ingest_time > source_time > 无法判断
```

### 时效状态
新增资讯 / 延迟获取 / 旧闻重抓 / 盘中异动解释 / 盘后预期 / 待盘面验证 / 已部分兑现 / 无法判断

### 来源类型
公告 / 新闻 / 政策 / 研报 / 复盘 / 小作文 / 自媒体 / 数据异动 / OCR低置信 / 时间节点 / 事件日历

### 主体类型
A股上市公司 / 港股上市公司 / 美股上市公司 / 海外公司 / 未上市公司 / 政策/监管机构 / 行业协会/交易所 / 产品/模型/项目 / 行业方向 / 宏观数据 / 未知主体

### 信息性质
事实 / 观点 / 推演 / 传闻 / 情绪 / 风险

### 催化类型
政策 / 涨价 / 订单 / 业绩 / 供需缺口 / 技术突破 / 国产替代 / 并购 / 澄清 / 监管 / 减持 / 情绪扩散

### 影响级别
板块级 / 个股级 / 情绪级 / 风险级

### 处理结果
主攻支撑 / 弹性支撑 / 观察 / 高影响低可信观察 / 小作文池 / 需要公告验证 / 降级 / 排雷 / 仅记录

### 证伪触发点
公司澄清 / 监管提示 / 交易所问询 / 董秘互动否认 / 权威媒体核查 / 客户供应商名单证伪 / 订单数据证伪 / 财报证伪 / 产品价格数据证伪 / 板块高开低走 / 核心股放量滞涨 / 资金撤退 / 市场风格切换

## 填写要求
1. 不再使用单独“时间”字段作为主字段，统一拆分为时间字段组。
2. `source_time` 来自原文、公告、新闻或数据源自带时间，缺失时不得猜测。
3. 原文只有日期时，只保留日期，不得硬补时分秒。
4. `ingest_time` 正式运行时应尽量精确到秒。
5. `analysis_time` 缺失时写 `null`。
6. `timezone` 默认写 `Asia/Shanghai`。
7. 时段属性优先根据 `ingest_time` 判断。
8. 如果单条资讯缺少 `ingest_time`，但文档级元数据有 `document_ingest_time`，可继承。
9. 来源类型必须按规则归类，不得写模糊描述。
10. H / L / C / W / I 与最终评级必须引用 [02_证据等级规则.md](./02_证据等级规则.md)。
11. 时间不会改变 H 证据硬度，只影响 C / W / I、时段属性、时效状态和处理结果。
12. 摘要只保留关键信息，不堆砌原文。
13. 对应方向可填写行业、题材、政策方向或市场主线。
14. 对应个股可填核心标的、跟风标的或留空。
15. 主体类型必须先区分 A 股、非 A 股、未上市、政策机构、产品模型、行业方向等不同主体。
16. 非 A 股主体不得直接当作 A 股股票；如需映射到 A 股，必须在风险点中写明“A股映射链待验证”。
17. 验证点必须可执行，不能写空泛表述。
18. 证伪触发点必须写明哪些事件会关闭叙事窗口。
19. 降级风险必须写明为何可能失真或失效。
20. 处理结果必须明确，不允许“待看”这类无裁判结论词。
21. 最终评级表示短线博弈影响力，不表示事实真实性。
