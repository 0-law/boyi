# 个股候选逻辑字段规范

## 文件定位
本文件规定完整资讯分析后，如何整理涉及到的 A 股个股候选逻辑。

本文件不是独立提示词。
本文件不是让 AI 再单独分析一遍文章。
本文件是字段规范。

## 正确流程
AI 一次性分析输入内容  
↓  
生成结构化中间结果  
↓  
根据本规范封装个股候选逻辑 CSV

## CSV 表头

```text
任务ID,股票,股票代码,所属方向,当日表现,核心逻辑浓缩,证据类型,兑现路径,验证点,风险降级原因,强度,写入状态,写入时间,失败原因
```

## 必备字段
- 任务ID
- 股票
- 股票代码
- 核心逻辑浓缩
- 写入状态
- 写入时间
- 失败原因

## 字段规则

### 1. 任务ID
用于标识一次完整分析任务。

### 2. 股票
只填写输入中明确出现的 A 股股票简称或 A 股上市公司。

### 3. 股票代码
股票代码必须尽量补齐。

填写规则：
1. 如果输入中明确提供股票代码，直接使用原文代码。
2. 如果输入中只提供股票名称，没有股票代码，AI 需要根据 A股股票名称补齐股票代码。
3. 股票代码必须使用 6 位 A股代码格式。
4. 如果代码以 0 开头，必须保留前导 0。
5. 如果股票名称存在歧义、无法唯一确认、疑似非 A股主体，股票代码留空。
6. 不得给美股、港股、海外公司、未上市公司、产品名、模型名、政策机构补 A股代码。
7. 不得为了补齐代码而把非 A股主体强行写成 A股股票。

示例：
- 江波龙 → 301308
- 绿的谐波 → 688017
- 深南电路 → 002916
- 中际旭创 → 300308

如无法确认唯一代码：
- 股票代码留空。

### 4. 所属方向
填写原文对应题材、行业、板块、逻辑方向。

### 5. 当日表现
只填写原文明确提到的盘面表现；未提及写 `未提供`。

### 6. 核心逻辑浓缩
必须来自原文，尽量 40 字以内，不写交易建议。

### 7. 证据类型
可用：

```text
公告 / 新闻 / 政策 / 研报 / 复盘 / 小作文 / 传闻 / 数据异动 / 盘面异动 / 业绩预告 / 风险提示 / 未知
```

可使用 `+` 组合。

### 8. 兑现路径
填写以下之一或组合：

```text
业绩兑现
订单落地
产品量产
客户验证
产能释放
价格上涨
政策细则落地
收入占比提升
板块持续性验证
盘面持续性验证
公告确认
未提供
```

### 9. 验证点
必须可执行，例如：
- 核验公告原文
- 核验业绩预告
- 核验订单金额
- 核验收入占比
- 核验客户关系
- 核验产品量产进度
- 核验板块持续性
- 核验是否旧闻重炒
- 核验是否存在风险提示

### 10. 风险降级原因
必须具体，例如：
- 未见公告验证
- 收入占比过低
- 纯概念蹭热点
- 来源不明
- 旧闻重炒
- 盘面高开低走
- 业绩兑现不及预期
- 客户关系未证实
- 非核心标的
- 板块高潮后分化

### 11. 强度
只使用：

```text
S / A / B / C / R
```

含义：
- `S`：强逻辑，证据较硬，兑现路径清楚，具备优先候选价值。
- `A`：逻辑较顺，有验证路径，但仍需进一步确认。
- `B`：有题材或方向关联，但证据或兑现路径偏弱。
- `C`：弱映射、低确定性、后排跟风、兑现路径模糊。
- `R`：风险、证伪、排雷、明显降级。

强度不是买卖评级，不表示可以买、会涨、适合追。

### 12. 写入状态
只使用：

```text
未写入
已写入
写入失败
跳过
```

AI 分析阶段默认写 `未写入`。

### 13. 写入时间
写入时间字段必须保留，但 AI 不负责填写。

默认留空。

不得填写 `null`。  
不得填写资讯发布时间。  
不得填写采集时间。  
不得填写分析时间。  
不得编造写入时间。

写入时间由后续入库程序在真实写库时自动填写。

CSV 中写入时间字段应输出为空字符串：

```csv
""
```

### 14. 失败原因
正常行写 `null`。

失败或跳过时填写，例如：
- 缺少股票名称
- 股票代码未确认
- 核心逻辑缺失
- 非A股主体
- 无法确认是否为个股
- 不是A股上市公司

## 关键边界
1. 不得机械抽取股票名，必须整理逻辑。
2. 不得重复分析整篇文章。
3. 个股 CSV 是完整资讯分析后的派生产物。
4. 不得编造股票代码。
5. 不得输出买卖建议、目标价、仓位建议。
