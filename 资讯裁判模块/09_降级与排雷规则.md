# 降级与排雷规则

## 目标
当资讯存在失真、冲突、夸大、过时、误传等情况时，统一执行降级与排雷，避免错误信息进入交易判断链。

## 核心原则
1. 低等级证据遇到高等级冲突，必须降级。
2. 来源不明的信息，宁可低估，不可高估。
3. 过期信息、二次加工信息、标题党信息，要降低影响权重。
4. 排雷不仅处理利空，也处理“伪利好”。
5. H1 小作文与 H5 公告冲突，优先采用 H5，原逻辑最终评级降为 R。
6. L0 逻辑冲突必须降级。
7. I0 负影响信息必须进入降级 / 排雷表。
8. W0：证伪窗口关闭，必须进入降级 / 排雷审查。

## 触发降级的主要场景

### 1. 与公告或监管信息冲突
- 传闻与公司澄清公告冲突
- 新闻描述与交易所文件冲突
- 纪要内容与正式披露冲突

处理原则：以更高等级证据为准，低等级内容直接降级。

### 2. 来源无法确认
- 只有截图
- 只有转述
- 没有发布时间
- 没有发布主体

处理原则：通常降为 H1 或 H0。

### 3. 信息存在二次加工
- 标题明显夸大
- 摘要删掉风险提示
- 自媒体只截取有利片段

处理原则：不得按原始材料同等级处理。

### 4. 信息已过时
- 旧闻重炒
- 旧纪要二次传播
- 老政策重新包装

处理原则：H 可保留，但 C、I 与最终评级应下降。

### 5. 仅有 OCR 或 AI 整理结果
- 图片识别文字无原图出处
- AI 摘要无原始链接
- 模型推演冒充事实结论

处理原则：默认低置信，不进入强证据链。

### 6. 业务与兑现路径不匹配
- 收入占比极低
- 仅送样
- 仅预研
- 未产品化
- 短期无收入利润

处理原则：L 降低，必要时直接判为 L0 或将最终评级降为 R。

### 7. 证伪窗口临近关闭
- 临近公司澄清
- 临近监管提示
- 临近交易所问询
- 临近公告发布
- 临近业绩披露
- 临近互动易回复

处理原则：W1 / W2 且临近澄清、监管、问询、公告、业绩披露、互动易回复时，必须降低最终评级。

### 8. 盘面证伪
- 板块高开低走
- 核心股放量滞涨
- 后排炸板
- 资金撤退
- 市场风格切换

处理原则：如果没有事实证伪，但盘面出现高开低走、核心股放量滞涨、后排炸板、资金撤退，也视为盘面证伪，必须降低 I 影响预期和最终评级。

## 排雷重点
- 澄清公告
- 减持公告
- 监管问询
- 财务造假风险
- 业绩低于预期
- 订单不及预期
- 题材证伪
- 伪龙头与蹭热点
- 收入占比极低
- 仅送样与预研映射
- 未产品化
- 短期无收入利润
- 板块高开低走
- 核心股放量滞涨
- 后排炸板
- 资金撤退

## 处理结果映射建议
- 高等级利空或证伪：排雷
- 低等级传闻被否认：降级
- H1 传闻被 H5 澄清公告否认：R
- L0 逻辑冲突：降级或排雷
- W0 证伪窗口关闭：降级或排雷
- W1 / W2 且临近澄清、监管、问询、公告、业绩披露、互动易回复：下调评级
- 盘面证伪导致 I 快速下降：降级观察或排雷
- I0 负影响信息：排雷
- 来源不明但有传播热度：小作文池
- 真实性不足但值得跟踪：观察
- 明显无效信息：仅记录

## 实操要求
每条被降级的资讯必须写清：
- 降级原因
- 冲突对象
- 当前保留价值
- 后续验证路径

## 最终目标
所有资讯都应经过降级与排雷筛选后，再进入“资讯裁判表”，从而把“市场听说”与“可用证据”彻底分开。
