# 采集输入格式规范

## 文件定位

本文档只规定采集端如何把原始资讯整理成 Markdown 格式输入。
本文档不负责资讯评级，不负责市场判断，不负责资金盘面分析，不负责交易决策。

资讯评级规则仍以下列文件为准：

- [02_证据等级规则.md](./02_证据等级规则.md)

## 时间元数据总原则

Markdown 格式资讯可以继续使用，但每条资讯应尽量带结构化时间元数据。

统一使用：

```text
ISO 8601 时间格式 + UTC+08 时区
```

A 股场景统一使用北京时间，例如：

```text
2026-07-03T09:35:12+08:00
```

默认时区：

```text
Asia/Shanghai
```

## 标准时间字段

每条资讯统一使用以下字段：

```yaml
source_time: 2026-07-03T09:12:00+08:00
source_time_precision: minute
source_session: 盘中
ingest_time: 2026-07-03T09:14:31+08:00
ingest_session: 交易日盘中
analysis_time: null
timezone: Asia/Shanghai
timeliness_status: 待盘面验证
```

字段含义：

- `source_time`：资讯原始发布时间。
- `source_time_precision`：原始发布时间精度。
- `source_session`：资讯原始发生时段。
- `ingest_time`：系统获得 / 抓取 / 导入该资讯的时间。
- `ingest_session`：系统拿到该资讯时所处时段。
- `analysis_time`：AI 实际裁判该资讯的时间，可由分析端填写。
- `timezone`：统一时区。
- `timeliness_status`：时效状态。

要求：

1. `source_time` 来自原文、公告、新闻或数据源自带时间。
2. `source_session` 反映资讯原始发生时段，不得被 `ingest_time` 覆盖。
3. `ingest_time` 来自采集端实际获取时间。
4. `ingest_session` 只反映系统何时拿到该资讯。
5. `analysis_time` 来自分析端实际执行时间。
6. `source_time` 缺失时不得猜测，写 `null`。
7. `analysis_time` 缺失时写 `null`。
8. `timezone` 默认写 `Asia/Shanghai`。
9. 不得编造时间，不得把自然语言时间硬改成伪精确时间。

---

## source_time_precision 枚举

固定使用以下枚举：

```text
minute
hour
daypart
date
unknown
```

说明：

- `minute`：精确到分钟。
- `hour`：精确到小时。
- `daypart`：只知道早间 / 晚间 / 夜间 / 上午 / 下午等时段。
- `date`：只有日期。
- `unknown`：未提供。

示例：

原文写“7月2日消息”：

```yaml
source_time: 2026-07-02
source_time_precision: date
```

原文写“7月2日晚”：

```yaml
source_time: 2026-07-02
source_time_precision: daypart
source_session: 盘后
```

原文写“7月2日7时46分”：

```yaml
source_time: 2026-07-02T07:46:00+08:00
source_time_precision: minute
source_session: 盘前
```

原文没有发布时间：

```yaml
source_time: null
source_time_precision: unknown
source_session: unknown
```

严禁把“7月2日消息”强行改成：

```yaml
source_time: 2026-07-02T09:00:00+08:00
source_time_precision: minute
```

这属于编造时间。

---

## source_session 枚举

固定使用以下枚举：

```text
盘前
盘中
盘后
夜间
非交易日
unknown
```

规则：

1. `source_session` 只根据原始发布时间和原文描述判断。
2. 不得因为系统在周末抓到资讯，就把原本工作日盘后消息改写成 `非交易日`。
3. 如果只有日期，没有足够依据判断具体时段，写 `unknown`。

---

## ingest_session 枚举

固定使用以下枚举：

```text
交易日盘前
交易日盘中
交易日盘后
非交易日
unknown
```

规则：

1. `ingest_session` 只看系统拿到资讯的时间。
2. `ingest_session` 可以是 `非交易日`，但这不改变 `source_session`。

---

## timeliness_status 枚举

固定使用以下枚举：

```text
实时
延迟获取
旧闻重抓
已过催化窗口
待盘面验证
unknown
```

规则：

1. `实时`：资讯发布时间与采集时间接近。
2. `延迟获取`：系统明显晚于原始发布时间才拿到资讯。
3. `旧闻重抓`：历史旧闻、旧纪要、旧政策被重新抓取。
4. `已过催化窗口`：事件日期已过，且短线催化窗口明显关闭。
5. `待盘面验证`：当前只拿到资讯文本，尚无盘面验证。
6. 没有盘面数据时，不得写 `已兑现`、`已部分兑现`。

## Markdown 单条资讯标准格式

````markdown
### 资讯 001：某公司中标公告

```yaml
source_time: 2026-07-02
source_time_precision: date
source_session: unknown
ingest_time: 2026-07-03T09:14:31+08:00
ingest_session: 交易日盘中
analysis_time: null
timezone: Asia/Shanghai
timeliness_status: 延迟获取
source_name: 某财经平台
source_url: null
collector: manual
```

7月2日消息，某公司披露中标某项目。
````

要求：

1. 每条资讯建议以 `### 资讯 001：标题` 开头。
2. YAML 元数据块必须紧跟标题。
3. 原文正文放在 YAML 元数据块之后。
4. 多条资讯按 `资讯 001 / 资讯 002 / 资讯 003` 顺序编号。
5. 标题只用于识别，不参与事实判断。

## Markdown 多条资讯文档标准格式

````markdown
# 2026-07-03 资讯采集

```yaml
document_id: jy-20260703-001
document_source: 某财经平台
document_ingest_time: 2026-07-03T09:30:00+08:00
document_ingest_session: 交易日盘中
timezone: Asia/Shanghai
collector: manual
```

## 1、机器人

### 资讯 001：某公司中标公告

```yaml
source_time: 2026-07-02
source_time_precision: date
source_session: unknown
ingest_time: 2026-07-03T09:30:00+08:00
ingest_session: 交易日盘中
analysis_time: null
timezone: Asia/Shanghai
timeliness_status: 延迟获取
source_name: 某财经平台
source_url: null
```

7月2日消息，某公司披露中标某项目。
````

要求：

1. 文档级元数据可以提供 `document_ingest_time` 与 `document_ingest_session`。
2. 如果单条资讯缺少 `ingest_time`，可继承 `document_ingest_time`。
3. 如果单条资讯缺少 `ingest_session`，可继承 `document_ingest_session`。
4. 如果单条资讯有自己的 `ingest_time / ingest_session`，优先使用单条资讯值。
5. 不允许用文档标题日期替代每条资讯的 `source_time`，除非原文明确该资讯发生在该日期。

## 采集端最小字段规范

每条资讯最少应提供：

```yaml
source_time: null
source_time_precision: unknown
source_session: unknown
ingest_time: 2026-07-03T09:30:00+08:00
ingest_session: 交易日盘中
analysis_time: null
timezone: Asia/Shanghai
timeliness_status: unknown
source_name: 某财经平台
source_url: null
raw_title: null
collector: manual
```

说明：

1. `ingest_time` 是系统知道这条资讯的时间。
2. `ingest_session` 是系统拿到资讯时所处时段。
3. `source_session` 与 `ingest_session` 必须拆开记录。
4. 后续结合市场反应时，应观察 `ingest_time` 之后的盘面变化，但不得反向改写 `source_session`。

---

## 采集端边界

采集端只负责：

1. 抓取原文。
2. 保留原文。
3. 记录原始发布时间。
4. 记录原始时段精度与原始发生时段。
5. 记录系统获取时间与系统获取时段。
6. 按 `Markdown + YAML` 格式输出。

采集端不负责：

1. H / L / C / W / I 评级。
2. 小作文判断。
3. 证伪窗口判断。
4. A股映射链判断。
5. 交易判断。
6. 买卖建议。
