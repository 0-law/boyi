# 资讯裁判样例输出

## 样例输入

以下为虚构示例，仅用于展示裁判方法：

1. `2026-07-03 08:20`，甲公司发布公告，确认中标某省级电网设备采购项目，金额明确。
2. `2026-07-03 08:45`，权威媒体发布新闻，称相关部委将于近期发布新型电网升级支持政策。
3. `2026-07-03 09:05`，财经平台快讯称乙公司可能受益于电网升级，但未引用公司公告。
4. `2026-07-03 09:18`，社群流传小作文，称甲公司已经进入海外大厂供应链，未见原始来源。
5. `2026-07-03 10:02`，某复盘文章称“市场认为乙公司会成为电网线核心弹性标的”。
6. `2026-07-03 10:30`，丙公司发布澄清公告，否认市场流传的“新增超大海外订单”。
7. `2026-07-03 10:50`，又一篇空泛小作文称乙公司“未来可能受益万亿市场”，但没有产品、客户、订单细节。

---

## 样例说明

- 样例一：硬公告，体现 `H5 + L5 + C4/C5 + I4/I5`，可评 `SS / SSS`。
- 样例二：正规新闻，体现 `H3/H4`，但如果仅是“将于近期发布”，信息性质应为预期，不应直接写成政策事实。
- 样例三：高影响小作文，体现 `H1 + L4 + C5 + W5 + I4`，可评 `S / SS观察`，但不能写成事实。
- 样例四：空泛小作文，体现 `H1 + L1/L2 + C3 + W5 + I1/I2`，不可因为不可证伪就升级。
- 样例五：复盘观点，体现 `H2`，只能作为市场共识样本。
- 样例六：澄清公告，体现 `H5 + W0 + I0`，与传闻冲突时，原传闻必须降为 `R`。
